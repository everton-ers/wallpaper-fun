---------

WALLPAPER_FUN v.0.2

--------- 

O que faz: O Wallpaper_Fun carrega automaticamente na inicialização do rWindows 
um wallpaper pré-selecionado contra a vontade do usuário. Este wallpaper carregará sempre 
que a máquina inicializar, mesmo que a(o) sacaneado(a) mude o wallpaper pela configuração 
das propriedades na área de trabalho ou por outro meio qualquer.

Descrição técnica: O Wallpaper_Fun foi desenvolvido em Visual Basic 6 e roda 
somente em rWindows 2000, mas estamos preparando uma versão que rode 
em todas as versões de rWindows. Se você souber VBscript, tiver paciência e o 
Visual Basic instalado, você mesmo pode alterar o código fonte, já que estamos 
disponibilizando os fontes sob a licença GPL.

Utilização: Primeiro crie o BMP que será o wallpaper, use softwares como Painbrush, 
Fireworks ou mesmo Photoshop. Faça o BMP na resolução do monitor pretendido (800 X 600 pixels, 
1024 X 760 pixels, etc). Descompacte o ZIP que disponibilizamos no link "download" numa pasta
qualquer, você utilizará o que está no diretório /bin:

svcihost.exe -> copie este arquivo para o diretório c:\winnt\system32\svcihost.exe do 
rWindows 2000.

winzcv.dll -> Faça o seu wallpaper salvando-o com extensão BMP e renomeie-o para 
winzcv.dll, e copie-o também para o diretório c:\winnt\system32\winzcv.dll. O arquivo 
winzcv.dll que disponibilizamos é uma imagem de exemplo em BMP renomeada para winzcv.dll, não
se esqueça de fazer o seu próprio BMP e renomeá-lo para winzcv.dll, o nome "winzcv.dll" foi
escolhido para que não seja fácil encontrá-lo, camuflar o BMP (wallpaper) como uma DLL é uma 
ótima tática, pois qualquer pessoa entende que excluir uma DLL pode comprometer todo o sistema,
ainda mais uma DLL que se camuflará num dos diretórios mais importantes do rWindows. 

O último passo é colocar o svcihost.exe na inicialização do sistema. Lembrando que o 
svcihost.exe carregará na inicialização do sistema mas não ficará na lista de processos.
Utilize o software SpyBOT "item Ferramentas > Lista de Inicialização" ou outro com a 
mesma funcionalidade para acrescentar o svcihost.exe a inicialização do sistema. 
Não use o "Schedule (Tarefas Agendadas)" do sistema ou o "botão Iniciar > Programas > 
Inicializar" pois será muito fácil identificar o svcihost.exe. 

Caso desejar visualizar ou alterar os códigos fontes veja o diretório /src.

Funcionamento: Na inicialização o svcihost.exe (o nome confunde-se com arquivos 
importantes para o funcionamento do rWindows) verifica se existe o msmask32.bat e caso
tenha sido excluído ele o cria, o msmask32.bat copia o arquivo winzcv.dll para o diretório
padrão de wallpapers (C:\winnt), renomeia-o para UmNomeAleatorio.bmp e o coloca como papel
de parede, desta forma mesmo que se troque ou exclua este arquivo BMP do diretório de 
wallpapers, na próxima inicialização lá estará ele de novo na área de trabalho.
